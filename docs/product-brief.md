# FundFit — Product Brief

## Thesis

FundFit helps small and mid-sized nonprofits that **already know their mission**
find, prioritize, and pursue funding opportunities more effectively. It is **not**
a generic "AI grant writer." It is a **funding strategy, funder-fit, and
application-readiness system**.

Most fundraising tools either (a) dump a giant searchable funder database on an
already-overwhelmed nonprofit, or (b) auto-generate grant prose. FundFit instead
answers the questions a time-strapped leader actually has and **shows its
reasoning** so the nonprofit can trust and override it.

## Core user

A nonprofit **executive director, development director, board member, or
volunteer fundraiser** with limited time who needs to answer:

- Which funders are worth pursuing?
- How should we frame our existing work for each funder?
- What materials are missing before we apply?
- What should we do this week to move funding forward?
- How do we reuse strong language across future grants?

## First test case

A **Common Courtesy-style mobility nonprofit**: coordinated, human-supported rides
(live phone dispatch, no app required) for seniors, people with disabilities,
medically vulnerable people, and others who can't drive. The app is intentionally
general enough to serve other nonprofits.

## Principles

1. **Funding cockpit, not a chatbot.** Scannable tables, briefs, and plans.
2. **Plain English. No overclaiming.**
3. **Recommend "Skip"** when a funder is a bad fit — protecting scarce time is the
   product.
4. **Every recommendation explains why.** Scoring is transparent and editable.
5. **AI/template output requires human review.** Nothing is auto-submitted.
6. **Easy onboarding.** Start from public-data prefill, not a blank form.
7. **Suggested, not asserted.** Every prefilled value shows a source type and
   confidence and can be accepted, edited, or removed.
8. **No invented grant facts.** All demo funders and figures are clearly labeled
   sample data.
9. **Respect mission boundaries.** The app surfaces what an org should *avoid*
   doing just to chase money.

## Feature map (MVP)

| Section | What it does |
|---------|--------------|
| Dashboard | Entry points + value props |
| Find My Nonprofit | Mock public-data search → suggested dossier review |
| Dossier | Structured profile; completeness + "most useful next field" |
| Documents | Simulated upload → extracted facts as suggested fields |
| Guided Interview | 7 plain-English questions → dossier content |
| Funding Frames | Mission retold in funder language (rule-based) |
| Funder Matches | 20+ scored, ranked funders with filters |
| Pursuit Brief | One-page, printable, board-ready action plan per funder |
| Grant Assets | Reusable template-generated language (editable) |
| Readiness | 8-category readiness score + highest-leverage actions |
| 90-Day Plan | Sequenced tasks with owner / due / status |

## Non-goals (for the MVP)

- Real funder data, live APIs, or auth.
- Auto-writing and submitting full grant applications.
- Acting as legal, tax, accounting, or guaranteed-outcome advice.

## Success criteria

A user can go end-to-end — search → review suggestions → dossier → frames →
ranked funders → pursuit brief → assets → readiness → 90-day plan — and come away
with a **defensible, explainable shortlist and a concrete next step**, in minutes.
